## Key Architectural Principles

### 1. Pixels as Code
Every instruction exists as RGBA pixel values in VRAM:
```
Pixel[0] = (0x30, 0x00, 0x05, 0x00)  # IMM32 R0, 5
         = (R,    G,    B,    A   )
         = (OP,   RD,   ARG, ARG )
```

### 2. Dual Execution Model
- **CPU Backend**: Python interpreter for development/debugging
- **GPU Backend**: SPIR-V for production performance

### 3. Frozen SPIR-V Layer
SPIR-V is the stable "microcode" - never changes.
All innovation happens in pxVM layer above it.

### 4. Universal Binary Compatibility
Translate anything to pxVM:
- x86/x86-64 â†’ pxVM
- ARM â†’ pxVM
- WASM â†’ pxVM
- RISC-V â†’ pxVM

### 5. Self-Modification by Design
Code can rewrite itself, watched in real-time as pixel changes.
LLM can optimize running programs.
